        public BigDataViewer(ArrayList<ConverterSetup> converterSetups,

                     AbstractSpimData<?> spimData,
                     int numTimepoints,
                     CacheControl cache,
                     String windowTitle,
                     ProgressWriter progressWriter,
                     ViewerOptions options)
        Parameters:
        converterSetups - list of ConverterSetup that control min/max and color of sources.
        sources - list of pairs of source of some type and converter from that type to ARGB.
        spimData - may be null. The AbstractSpimData of the dataset (if there is one). If it exists, it is used to set up a "Crop" dialog.
        numTimepoints - the number of timepoints in the dataset.
        cache - handle to cache. This is used to control io timing.
        windowTitle - title of the viewer window.
        progressWriter - a ProgressWriter to which BDV may report progress (currently only used in the "Record Movie" dialog).
        options - optional parameters.

        public static org.scijava.ui.behaviour.io.InputTriggerConfig getInputTriggerConfig(ViewerOptions options)
        If options doesn't define a InputTriggerConfig, try to load it from files in this order:
        1. "bdvkeyconfig.yaml" in the current directory.
        2. ".bdv/bdvkeyconfig.yaml" in the user's home directory.
        3. legacy "bigdataviewer.keys.properties" in current directory (will be also written to "bdvkeyconfig.yaml").
