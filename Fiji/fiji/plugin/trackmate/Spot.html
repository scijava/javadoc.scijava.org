Class Spot

  • java.lang.Object
    • net.imglib2.AbstractEuclideanSpace
      • fiji.plugin.trackmate.Spot
  • All Implemented Interfaces:
    Comparable<Spot>, net.imglib2.EuclideanSpace, net.imglib2.RealLocalizable

    public class Spot
extends net.imglib2.AbstractEuclideanSpace
implements net.imglib2.RealLocalizable, Comparable<Spot>
    A RealLocalizable implementation, used in TrackMate to represent a detection.

    On top of being a RealLocalizable, it can store additional numerical named features, with a Map-like syntax. Constructors enforce the specification of the spot location in 3D space (if Z is unused, put 0), the spot radius, and the spot quality. This somewhat cumbersome syntax is made to avoid any bad surprise with missing features in a subsequent use. The spot temporal features (FRAME and POSITION_T) are set upon adding to a SpotCollection.

    Each spot received at creation a unique ID (as an int), used later for saving, retrieving and loading. Interfering with this value will predictively cause undesired behavior.

    Author:
    Jean-Yves Tinevez <jeanyves.tinevez@gmail.com> 2010, 2013

      • normalizeDiffTo

        public double normalizeDiffTo(Spot s,
                              String feature)
        Returns the absolute normalized difference of the feature value of this spot with the one of the given spot.

        If a and b are the feature values, then the absolute normalized difference is defined as Math.abs( a - b) / ( (a+b)/2 ).

        By construction, this operation is symmetric ( A.normalizeDiffTo(B) = B.normalizeDiffTo(A)).

        Will generate a NullPointerException if one of the spots does not store the named feature.

        Parameters:
        s - the spot to compare to.
        feature - the name of the feature to use for calculation.
        Returns:
        the absolute normalized difference feature value.
