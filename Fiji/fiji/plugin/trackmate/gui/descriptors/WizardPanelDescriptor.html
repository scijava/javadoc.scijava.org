fiji.plugin.trackmate.gui.descriptors

Interface WizardPanelDescriptor

  • All Known Implementing Classes:
    ActionChooserDescriptor, ConfigureViewsDescriptor, DetectionDescriptor, DetectorChoiceDescriptor, DetectorConfigurationDescriptor, GrapherDescriptor, InitFilterDescriptor, LoadDescriptor, LoadICYTrackPlugIn_, LoadTrackMatePlugIn_, LogPanelDescriptor, SaveDescriptor, SomeDialogDescriptor, SpotFilterDescriptor, StartDialogDescriptor, TrackerChoiceDescriptor, TrackerConfigurationDescriptor, TrackFilterDescriptor, TrackingDescriptor, ViewChoiceDescriptor

    public interface WizardPanelDescriptor
    A base descriptor class used to reference a Component panel for the Wizard, as well as provide general rules as to how the panel should behave.

    Four methods are used to plug actions when the user navigate through the GUI. When the user "navigates forward" - that is when the GUI flow to go in the classical direction - the following chain of events are triggered upon pressing 'Next':

    1. The descriptor currently displayed executes aboutToHidePanel().
    2. The identity of the next descriptor is determined.
    3. This descriptor executes aboutToDisplayPanel().
    4. The descriptor component is displayed on the GUI.
    5. The descriptor executes displayingPanel().

    When the user navigates backward, the following chain of events are triggered upon pressing 'Previous':

    1. The identity of the descriptor to navigate to is determined.
    2. The descriptor executes comingBackToPanel().
    3. The descriptor executes displayingPanel().
    4. The descriptor component is displayed on the GUI.

    Therefore, only displayingPanel() is executed in both directions, and is used typically to set up and refresh what is displayed on the descriptor component.

        void displayingPanel()
        This method is used to perform functionality when the panel itself is displayed. It is the responsibility of concrete implementations to set the state of the GUI, such as the 'Next' button enabled/disabled. The 'Next' button is disabled by the GUI at the start of each 'Next' action. Concrete implementations must re-enable it before this method returns. For instance, you want to disable the 'Next' button while an intense calculation is ongoing, then re-enable it.
