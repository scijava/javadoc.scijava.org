    public class FastIntegralImg<R extends RealType<R>,T extends NativeType<T> & NumericType<T>>
extends ImgProxy<T>
    Integral image class tweaked for maximum performance. For a 512x512, 2d UnsignedByteType image with a target integral image of LongType: 79 ms for the IntegralImg from the algorithms package. 72 ms for the generic nd method implemented here. 52 ms for the generic 2d method implemented here. 13 ms for special-purpose 2d method when both images are backed up by arrays. There are special-purpose methods for 2d images that are backed up by arrays, and whose dimensions are under Integer.MAX_VALUE, for the following type combinations: 1) Input Img of type UnsignedByteType and integral image of type UnsignedIntType; 2) Input Img of type UnsignedByteType and integral image of type UnsignedLongType; 3) Input Img of type UnsignedShortType and integral image of type UnsignedIntType; 4) Input Img of type UnsignedShortType and integral image of type UnsignedLongType; 5) Input Img of type FloatType and integral image of type DoubleType; For these special-purpose methods, the converter is ignored.
    Author:
    Albert Cardona
