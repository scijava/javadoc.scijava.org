    public abstract class AbstractRenderer<A>
extends Object
implements Renderer<A>
    Render source data into a BufferedImage and provide this to a RenderTarget. Handle repaint requests by sending them to a PainterThread.

    The source data is rendered given a viewer transform (mapping global to screen coordinates). Often there is an additional source transform, mapping source to global coordinates. The transformation to apply to the source data then is a composition of this source transform (source to global) and the viewer transform (global to screen).

      • requestRepaint

        public void requestRepaint()
        Request a repaint of the display from the painter thread. The painter thread will trigger a paint as soon as possible (that is, immediately or after the currently running paint has completed).

        All repaint request should be directed through here, usually not to PainterThread.requestRepaint() directly. The reason for this is, that derived classes (i.e., MultiResolutionRenderer) may choose to cancel the on-going rendering operation when a new repaint request comes in.
