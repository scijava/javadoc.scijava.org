extends AbstractFilter<T>
implements MasterFilter<T>
    Helper class for MasterFilter implementations. Takes the place of an abstract superclass for containing boilerplate code.

    As all MasterFilter implementations are also Filters, it makes sense for concrete implementations of the former to be free to inherit from abstract implementations of the latter. Thus, any boilerplate code that would be common to all MasterFilter implementations, regardless of their wrapped type, must go in a helper class to avoid multiple inheritance conflicts.

    On construction, all Filter classes annotated with Plugin are discovered by the SciJava Common framework. All items whose wrappedClass() return the same class as was used to construct this MasterFilterHelper will be added to the list of filters for future use. Filters are initially enabled or disabled based on their isDefaultEnabled() annotation return value.

    Filters can be enabled or disabled using the enable(Class) and disable(Class) methods. This process can be done in any order, as execution ordering of enabled filters is maintained automatically per their natural order.
