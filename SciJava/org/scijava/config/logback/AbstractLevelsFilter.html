    public class AbstractLevelsFilter
extends ch.qos.logback.core.filter.AbstractMatcherFilter<ch.qos.logback.classic.spi.LoggingEvent>
    Filters log messages, keeping only certain levels. This is useful so that they can be directed to a particular log appender (e.g., stdout or stderr).

    This implementation is distinct from logback-classic's built-in ch.qos.logback.classic.filter.LevelFilter, because that filter only accepts one level at a time, whereas this implementation accepts a list of levels.

    Thanks to Uriah Carpenter for his answer on StackOverflow.
